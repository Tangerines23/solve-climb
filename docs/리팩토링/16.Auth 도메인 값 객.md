너는 지금부터 우리 프로젝트의 '도메인 모델링 전문가'이다.

Auth 도메인의 Email과 Tier 데이터에 값 객체(Value Object)와 스마트 생성자를 도입할 것이다.

가장 중요한 조건은 '익명 로그인(Anonymous User)' 상태를 우아하게 처리(Null Object Pattern)하는 것이다.

1. Email 값 객체 구현 지시사항

파일 위치: src/features/auth/domain/Email.ts

익명 유저는 이메일이 null일 수 있다. 스마트 생성자 create(value: string | null | undefined)는 값이 없을 경우 에러를 뱉지 말고, { success: true, data: null } 또는 명시적인 익명 이메일 객체를 반환해야 한다.

값이 존재할 경우, 정규식(또는 Zod)을 사용하여 철저하게 이메일 형식을 검증하라.

2. Tier 값 객체 구현 지시사항

파일 위치: src/features/auth/domain/Tier.ts

티어는 원시 숫자(number)가 아니라, 특정 비즈니스 룰을 가진다.

익명 유저 대응: 스마트 생성자에 null이 들어오면, UI에서 에러가 나지 않도록 Tier.UNRANKED (또는 isUnranked: true 속성을 가진)라는 특별한 정적 인스턴스(Null Object Pattern)를 반환하라.

UI 컴포넌트에서는 tier.value를 직접 찌르기보다 tier.getDisplayName() (예: "브론즈", "등급 없음") 같은 포맷팅 메서드를 호출할 수 있도록 객체 내부에 행위를 캡슐화하라.

3. 실행 단계

위 조건에 맞게 Email.ts와 Tier.ts의 클래스 코드를 먼저 작성하여 나에게 보여주어라.

코드가 승인되면, 기존 전역 상태(예: useUserStore)와 브릿지 훅(useAuthBridge)이 이 스마트 생성자를 통과하도록 리팩토링할 계획을 세워라.
