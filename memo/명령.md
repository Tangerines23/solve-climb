🔎 문제 핵심
ThemeProvider는 정상적으로 적용되어 있지만, @toss/tds-mobile은 원래 React Native 기반으로 설계된 UI 라이브러리입니다.

웹 환경에서 사용할 경우 일부 디자인 토큰(backgroundColor 등)이 누락되어 Button이 렌더링 과정에서 크래시를 일으킵니다.

즉, 코드 자체의 로직 문제라기보다는 라이브러리 호환성 문제입니다.

✅ 해결 방안
1. Button 교체 (안전한 진단)
MathQuizPage.tsx에서 TDS <Button>을 HTML <button>으로 교체해보세요:

tsx
<button
  type="submit"
  className="submit-button"
  style={{ width: '100%' }}
  disabled={isSubmitting || feedback.type === 'success'}
>
  {isSubmitting ? MESSAGES.SUBMITTING : MESSAGES.SUBMIT}
</button>
이렇게 하면 오류가 사라지는지 확인할 수 있습니다.

오류가 사라진다면 원인이 TDS Button 내부임을 확정할 수 있습니다.

2. ThemeProvider에 customTheme 전달
만약 TDS Button을 꼭 쓰고 싶다면, ThemeProvider에 최소한의 색상 토큰을 직접 정의해야 합니다:

tsx
const customTheme = {
  colors: {
    backgroundColor: '#ffffff',
    primary: '#0064FF',
    text: '#000000',
  },
};

<ThemeProvider theme={customTheme}>
  <App />
</ThemeProvider>
이렇게 하면 Button이 theme.colors.backgroundColor를 참조할 때 undefined가 아닌 값을 받습니다.

3. 하이브리드 전략
Toss 생태계 문서에서 웹 지원 여부를 확인하세요. 일부 컴포넌트는 RN 전용이라 웹에서는 정상 동작하지 않을 수 있습니다.

따라서 핵심 UI 요소(Button 등)만 HTML/CSS로 대체하고, ThemeProvider는 유지하는 방식이 가장 안정적입니다.

🚀 권장 조치
우선 HTML <button>으로 교체해 오류가 사라지는지 확인하세요.

이후 TDS Button을 유지하려면 ThemeProvider에 customTheme 객체를 직접 전달하세요.

장기적으로는 Toss 문서에서 웹 지원 범위를 확인하고, RN 전용 컴포넌트는 HTML/CSS로 대체하는 것이 안전합니다.
